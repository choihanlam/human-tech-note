데이터를 많이 모으는 시대에서, 안전하게 연결하는 시대로

2024 데이터경제 컨퍼런스로 읽는 데이터 활용·보호·서비스 경쟁

2026년 7월 13일 (월)약 11분#50 / 75

들어가며

생성형 AI가 빠르게 확산되면서 기업은 모델의 성능과 업무 적용 가능성에 주목했다. 그러나 실제 서비스를 구축하는 단계에서는 모델보다 먼저 데이터의 품질, 권리, 공유 범위와 책임 문제가 드러났다.

2024년 데이터 경제의 핵심 질문은 “데이터를 보유하고 있는가”에서 “데이터를 신뢰할 수 있는 방식으로 연결하고 활용할 수 있는가”로 이동했다.

2024년 1월 제4회 데이터 경제 컨퍼런스는 생성형 AI와 데이터 기반 혁신의 기회·위험을 다뤘다. 같은 해 12월 제5회 컨퍼런스는 전자상거래, 금융, 에너지와 헬스케어에서 데이터가 서비스와 경쟁의 자원으로 전환되는 현상을 논의했다.

1. 2024년 1월: 생성형 AI의 기회와 위험

제4회 컨퍼런스의 주제는 「혁신, 시장과 데이터: 사회 발전의 동력」이었다. 생성형 AI, 빅데이터와 머신러닝, 정밀의료, 공공·시장 데이터 기반과 데이터 활용·보호가 주요 의제로 제시됐다.

이 시기의 논의는 AI를 도입할 것인지 여부를 넘어, AI가 사용하는 데이터의 출처와 품질을 어떻게 확인하고 개인정보·저작권·기밀정보의 위험을 어떻게 관리할 것인지에 초점을 맞췄다.

Source

데이터 출처와 신뢰성

AI가 참고한 데이터와 문서의 출처를 확인할 수 있어야 결과의 신뢰성을 평가할 수 있다.

Quality

데이터 품질과 최신성

중복되거나 오래된 데이터, 부서마다 다른 정의는 AI의 분석과 답변을 불안정하게 만든다.

Rights

활용 권리와 동의 범위

데이터에 접근할 수 있다는 사실과 이를 AI 학습·분석·외부 서비스에 사용할 수 있다는 것은 다른 문제다.

Accountability

결과와 책임

AI 결과가 잘못됐을 때 개발자, 운영기관, 데이터 제공자와 최종 의사결정자의 책임을 구분해야 한다.

2. 2024년 12월: 데이터 거래와 서비스 경쟁

제5회 컨퍼런스의 주제는 「데이터 기반 서비스 산업: 데이터 거래와 경쟁」이었다. 논의의 중심은 데이터 활용의 원칙에서 실제 산업과 비즈니스 모델로 이동했다.

산업데이터 활용 방향핵심 쟁점
전자상거래검색·구매·결제·후기 데이터를 추천, 광고와 상품운영에 활용플랫폼 데이터 집중, 입점업체와의 공정한 경쟁
금융임베디드 금융, Banking-as-a-Service, 기술평가정보개인신용정보, API 보안, 사고 발생 시 책임
에너지수요 예측, 설비 운영, 에너지 절감 서비스데이터 표준과 실시간 품질, 기관 간 연계
헬스케어임상·유전체·건강 데이터를 활용한 개인화 서비스민감정보, 연구윤리, 품질과 설명 가능성

데이터는 더 이상 내부 보고서를 만드는 보조자료에 머물지 않는다. 상품 추천과 가격, 금융 기능, 건강관리와 산업 운영을 구성하는 생산요소가 됐다. 동시에 데이터를 많이 보유한 플랫폼의 우위가 시장의 진입장벽이 될 수 있다는 경쟁정책의 문제도 커졌다.

3. 2024년 데이터 경제의 세 가지 변화

첫째, 모델 중심에서 데이터 기반 중심으로

AI 모델을 도입하는 것만으로는 경쟁력이 만들어지지 않는다. 조직이 보유한 데이터를 찾고, 의미와 품질을 표준화하고, 사용할 수 있는 권리와 보안조건을 명확히 해야 AI가 실제 업무에 적용될 수 있다.

둘째, 보호에서 ‘안전한 활용’로

데이터 보호는 여전히 필수적이지만, 활용을 완전히 차단해서는 데이터의 경제적·사회적 가치가 발생하지 않는다. 필요한 주체가 정해진 목적과 권한 안에서 데이터를 사용할 수 있도록 설계하는 것이 중요해졌다.

활용과 보호의 관계
보호되지 않는 데이터는 신뢰를 얻기 어렵고, 활용되지 않는 데이터는 가치를 만들기 어렵다.

셋째, 데이터가 시장경쟁의 조건으로

데이터가 축적될수록 서비스가 개선되고, 개선된 서비스가 다시 이용자와 데이터를 끌어오는 순환구조가 형성된다. 따라서 데이터 접근권과 공유조건, 가치배분은 개인정보보호만의 문제가 아니라 산업경쟁과 혁신정책의 문제로 확대됐다.

4. 데이터 거버넌스를 쉽게 설명하면

데이터 거버넌스는 데이터를 한곳에 저장하는 시스템이 아니다. 조직이 데이터를 활용하기 위해 필요한 의사결정권, 역할, 기준과 책임을 정하는 운영체계다.

누가 어떤 데이터를 관리하고, 어떤 목적으로 사용할 수 있으며, 품질과 활용 결과에 누가 책임질 것인가?
Owner

소유와 책임

데이터를 생성·관리·활용하는 조직의 역할과 최종 책임자를 구분한다.

Classify

분류와 위험도

개인정보, 민감정보, 영업비밀, 공개정보 등 특성과 위험 수준에 따라 기준을 달리한다.

Access

접근과 활용권한

조회 권한과 분석·결합·AI 학습에 사용할 수 있는 권한을 구분한다.

Share

공유와 거래

외부 제공 시 목적, 범위, 기간, 품질, 보안조치와 사고책임을 계약과 절차로 정의한다.

Trace

기록과 추적

누가 언제 어떤 데이터를 사용했고 어떤 결과를 만들었는지 로그와 증적으로 남긴다.

5. 기업이 점검해야 할 질문

  • 우리 조직의 핵심 데이터가 어디에 있고 누가 책임지는가?
  • 데이터의 의미와 품질 기준이 부서와 시스템 사이에서 일치하는가?
  • 해당 데이터를 분석, AI 학습과 외부 서비스에 사용할 권리가 있는가?
  • 협력사나 플랫폼과 공유할 때 목적·기간·재사용·폐기 기준이 정의돼 있는가?
  • 데이터 활용의 성과뿐 아니라 개인정보 침해, 편향과 보안위험도 측정하는가?

마치며

2024년 두 차례 컨퍼런스는 데이터 경제의 논의가 생성형 AI의 가능성과 위험에서 데이터 기반 서비스와 산업경쟁으로 구체화되는 과정을 보여줬다.

데이터 경제의 경쟁력은 가장 많은 데이터를 가진 조직에만 주어지지 않는다. 데이터의 의미와 품질을 관리하고, 권한과 책임을 명확히 하며, 외부와 안전하게 연결할 수 있는 조직이 데이터를 실제 가치로 전환할 수 있다.

2024년은 데이터를 많이 모으는 경쟁에서, 데이터를 안전하게 연결하고 서비스의 가치로 전환하는 경쟁으로 이동한 시기였다.

그리고 다음 질문이 남는다. 데이터를 안전하게 활용할 기반을 마련했다면, AI는 연구와 혁신의 방식 자체를 어떻게 바꾸고 있으며 그 성과는 어떻게 측정해야 할까?
